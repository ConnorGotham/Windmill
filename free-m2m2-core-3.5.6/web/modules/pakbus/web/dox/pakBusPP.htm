PakBus Data Point Help

A PakBus data point stores data from a column in a datalogger's table. There are 2 types of points Historical and Realtime.

Historical Points

Historical points access multiple rows from a datalogger's table. By choosing a Historical record start position all records including that record number are retrieved and stored in Mango. This allows previously recorded data to be retrieved. As the records are stored in Mango the latest record id is stored so that on the next poll only new records are retrieved. Note that historical points are grouped together per table so that only 1 request per table is made for each data source poll.

Historical points allow the tracked record id to be reset at any time while the Data Point is running.

Realtime Points

Realtime points gather the latest record from the table by only requesting one cell from the table. This is more efficient that a historical request but only allows access to the most recent value.

Common Settings

  • Device Name: the name of the device these points belong to, user definable and usually the name of the data source is used.
  • Name: the name of the point for use as an identifier in lists of points.
  • Settable: whether the point can be manually set or not.
  • Data type: the data type of the point. See below for details.
  • Logger address: the address of the datalogger for which this point belongs to.
  • Table name: the name of the table in the datalogger for which this point belongs to.
  • Column name: the name of the column in the table of the datalogger for which this point belongs to.
  • Array index: a 0 based index for the item in an array, leave as 0 for scalar values.
  • Collection type: Historical or Realtime data collection.

Historical Settings

  • Historical record start position: the initial position to start collecting data from. This can be reset while the point is running by entering a value in the field and clicking the Reset tracked starting record
