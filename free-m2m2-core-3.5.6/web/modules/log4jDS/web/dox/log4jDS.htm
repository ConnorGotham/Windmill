Overview

The Log4j data source is used to store log messages from the Log4j logging system. When the data source starts an asynchronous logging appender is added to the root logger to intercept messages. This happens in a separate thread as to not slow down the system.

Configuration

Every data source requires a Name, which can be any description. The Message repeat interval (ms) is a delay on handling the same log message as the preceding within the interval. If a different log message occurs or the interval elapses after the first message, the second identical message will be handled by any points it matches. This is primarily to prevent the simplest log message infinite loops.
