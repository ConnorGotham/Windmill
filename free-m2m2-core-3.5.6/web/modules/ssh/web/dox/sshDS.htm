Configuring your system for SSH Public Key Based Authentication

Linux Example (OpenSSH)

  • login to the mango server
  • execute ssh-keygen -t rsa Do not enter a passphrase
  • Copy the generated public key to the server you wish to login to at ./ssh/name_of_public_keyfile Example: scp ~/.ssh/id_rsa.pub user@host.org:.ssh/authorized_mango_keys
  • login to the server you plan to ssh into with Mango
  • if the file ~/.ssh/authorized_keys does not exist create it
  • Concatenate the newly copied public key to the list of authorized keys by: cat ~/.ssh/authorized_mango_keys >> ~/.ssh/authorized_keys
  • ensure correct permissions on all files by executing chmod 600 on them
  • if you need to change the passphrase on the public key workstation use ssh-keygen -p

The default location of the authorized keys if you're running OpenSSH should be /root/.ssh/authorized_keys, Dropbear should be /home/root/.ssh/authorized_keys, but your distribution may vary.
