Overview

This page provides a fast way to apply set and read permissions to data points. Data point groups can be filtered by creating filters to find desired points.

Creating a Filter

Filters are applied if the Filter State column is set to anything besides Disabled. Multiple filters can be applied at the same time. Items in the results will match all enabled filters. Filter allow pattern matching of strings and can use either % or * as wildcards, as well as a prefix of '^' for NOT, all other characters are treated as is. Using the value null will match null values, but using the value *null* will match a string that contains the text 'null'.

Each Filter row can be applied as a boolean And or a boolean Or by setting the Filter State accordingly. And operations require a point to match all of the required filter parameters of that filter. Or operators allow matching only one of the enabled filter parameters. Additionally the filter row can be applied using inverse criteria by selecting Not And or Not Or.

  • Disabled no columns of the filter are applied
  • And requires the point to match all columns of the filter
  • Or requires the point to match one column of the filter
  • Not And applies a not condition to all columns, then requires the point to match all columns of the filter
  • Not Or applies a not condition to all columns, then requires the point to match one column of the filter

Each filter parameter can be disabled or enabled for the current request by checking the check box in the column header. Disabling the parameter excludes it from the query entirely. To edit the value of the filter parameter, click in the cell for the corresponding filter and start typing. Hitting enter will close the editor and update the filter results. Filter cells cannot be edited if their column is disabled.

Note: When applying a Not And or Not Or condition be careful when using the '^' prefix for NOT as it will be ignored and not result in a double not condition.

Applying Permissions

Permissions can be applied by adding one or more groups separated by commas into the box and then clicking Apply permissions. To clear all permissions for the points in view click the Clear permissions button. Permissions will not be applied twice and trailing commas will be removed. The total number of data points whose permissions are modified will be returned. For example if your filter shows 10 points but 5 of them already have the permission 'user' then applying the permission 'user' to that filter group will only modify the 5 points without that permission.
